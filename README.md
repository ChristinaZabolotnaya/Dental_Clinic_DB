# DentalClinic OLAP Project

## Описание проекта
Данный проект представляет собой информационно-аналитическую систему для анализа деятельности стоматологической клиники, построенную на принципах OLAP.

Проект включает:
- загрузку данных из CSV-файлов
- построение хранилища данных (Data Warehouse)
- создание многомерного куба (OLAP)
- анализ данных по различным измерениям



## Установка

### . Требования
- Python 3.12
- Microsoft SQL Server
- Visual Studio 2022 (SSAS)



### . Установка зависимостей


pip install pandas pyodbc

## Архитектура проекта

### Хранилище данных (SQL Server)
База данных: `DentalClinicDB`

#### Таблицы измерений (Dimensions):
- DimPatients
- DimDoctors
- DimDate
- DimVisitType
- DimDiagnoses

#### Таблица фактов (Fact):
- FactVisits


## ETL-процесс

Загрузка данных реализована с помощью Python:

- библиотека `pandas` — чтение CSV
- библиотека `pyodbc` — загрузка в SQL Server

### Основные этапы:
1. Чтение CSV-файлов
2. Очистка и преобразование данных
3. Загрузка в таблицы базы данных


## OLAP-куб (SSAS)

Куб построен на основе таблицы фактов `FactVisits`.

### Меры (Measures):
- Profit (Прибыль)
- ServiceCost (Доход)
- VisitCount (Количество визитов)
- AvgTreatmentCost (Средняя стоимость лечения)


### Измерения (Dimensions):
- Пациенты (DimPatients)
- Врачи (DimDoctors)
- Даты (DimDate)
- Тип визита (DimVisitType)
- Диагнозы (DimDiagnoses)

## Примеры аналитики

С использованием куба выполнены следующие анализы:

- Количество визитов по диагнозам и возрасту
- Прибыль по врачам
- Динамика визитов по датам
- Доход по типам услуг


## Используемые технологии

- Microsoft SQL Server
- SQL Server Analysis Services (SSAS)
- Python 3.12
- pandas
- pyodbc
- Visual Studio 2022


## Как запустить проект

1. Создать базу данных `DentalClinicDB` в SQL Server
2. Выполнить SQL-скрипты создания таблиц
3. Запустить Python-скрипт загрузки данных
4. Открыть проект в Visual Studio
5. Настроить Data Source
6. Выполнить Deploy куба
7. Открыть вкладку Browser для анализа
